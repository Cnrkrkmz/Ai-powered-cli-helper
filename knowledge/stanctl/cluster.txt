# stanctl — Cluster management (up, down, create, delete, backup, restore)

[up]
Creates the cluster or starts a stopped cluster.
Usage: stanctl up [flags]
Required (initial setup): --core-base-domain, --download-key, --sales-key, --unit-tenant-name, --unit-unit-name, --unit-initial-admin-password, --unit-agent-key
Install type: --install-type demo | production
Automatic TLS: --core-tls-generate-cert
Manual TLS: --core-tls-crt <file> --core-tls-key <file>
Air-gapped mode: --air-gapped
Skip upgrade prompt: --force-auto-upgrade
Skip preflight check: --skip-preflight-check
Skip cluster creation (update backend only): --skip-cluster-creation
Single→multi-node migration: --multi-node-migrate --multi-node-enable --multi-node-ips <ips>
Multi-node: --multi-node-enable --multi-node-ips 192.168.1.1,192.168.1.2,192.168.1.3
Disable HPA: --core-enable-hpa=false
Override volume directories:
  --volume-data <path>        (default: /mnt/instana/stanctl/data)
  --volume-metrics <path>     (default: /mnt/instana/stanctl/metrics)
  --volume-analytics <path>   (default: /mnt/instana/stanctl/analytics)
  --volume-objects <path>     (default: /mnt/instana/stanctl/objects)
Proxy: --core-proxy-host, --core-proxy-port, --core-proxy-user, --core-proxy-password
SMTP: --core-smtp-host, --core-smtp-port (587), --core-smtp-from, --core-smtp-user, --core-smtp-password

[down]
Shuts down the cluster. Cleans up containers and networking. Data is not deleted.
Usage: stanctl down
Global flags apply (-q, -d, -t).

[cluster-create]
Creates only the K8s cluster (does not install backend).
Usage: stanctl cluster create [flags]
Alias: stanctl cl c
Multi-node: --multi-node-enable --multi-node-ips <ip1,ip2,ip3>
Air-gapped: --air-gapped --cluster-data-dir <dir>
Skip preflight: --skip-preflight-check

[cluster-delete]
Deletes the K8s cluster. Also deletes data by default.
Usage: stanctl cluster delete [--keep-data]
Alias: stanctl cl delete | del | rm | d
Preserve data: --keep-data

[cluster-backup]
Backs up the cluster.
Usage: stanctl cluster backup [-o <dir>]
Output directory: -o, --output-dir (default: current directory)

[cluster-restore]
Restores the cluster from a backup.
Usage: stanctl cluster restore -f <backup-file> [flags]
Required: -f, --file <backup-archive>
Multi-node restore: --multi-node-enable --multi-node-ips <ips>
Air-gapped: --air-gapped --cluster-data-dir <dir>
