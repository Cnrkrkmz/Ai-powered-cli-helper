# TANIM: Instana Self-Hosted Standard Edition backend ve cluster yönetim CLI aracı
# ANAHTAR KELİMELER: instana, backend, cluster, datastore, volume, object, analytics, metrics, data, migrate, license, agent, air-gapped
# stanctl komutları için term-ai bilgi tabanı
# Instana Self-Hosted Standard Edition yönetim CLI'ı
#
# Aliases (kısa kullanım):
#   agent       → ag
#   air-gapped  → agap
#   backend     → be
#   benchmark   → bm
#   cluster     → cl
#   datastores  → ds
#   dump        → dm
#   license     → lc
#   migrate     → mg
#   versions    → v
#
# Global flag'ler her komutta kullanılabilir:
#   -d, --debug               Verbose logging + manifest çıktısı
#   -q, --quiet               Tüm onay sorularını otomatik "evet" kabul et
#   -t, --timeout duration    Maksimum çalışma süresi (varsayılan: 30m0s)
#   --env-file string         Flag değerlerini dosyadan yükle (varsayılan: ".env")
#   --skip-resource-check     Node kapasite kontrolünü atla

[general]
stanctl, Instana Self-Hosted Standard Edition'ı yönetmek için kullanılan CLI aracıdır.
Her komut için STANCTL_ prefix'li environment variable karşılığı vardır (flag --download-key → env STANCTL_DOWNLOAD_KEY).
--instana-release ve --instana-version flag'leri birbirini dışlar, ikisini aynı anda kullanma.
Varsayılan registry URL: artifact-public.instana.io (kullanıcı: _, şifre: download-key)
Varsayılan Helm repo: https://artifact-public.instana.io/artifactory/rel-helm-customer-virtual
Volume varsayılan yolları:
  --volume-data      /mnt/instana/stanctl/data
  --volume-metrics   /mnt/instana/stanctl/metrics
  --volume-analytics /mnt/instana/stanctl/analytics
  --volume-objects   /mnt/instana/stanctl/objects

[up]
Cluster'ı oluşturur veya durdurulmuş cluster'ı başlatır.
Kullanım: stanctl up [flags]
Zorunlu (ilk kurulumda): --core-base-domain, --download-key, --sales-key, --unit-tenant-name, --unit-unit-name, --unit-initial-admin-password, --unit-agent-key
Install type: --install-type demo | production (varsayılan belirsiz, production için belirt)
Multi-node: --multi-node-enable --multi-node-ips 192.168.1.1,192.168.1.2,192.168.1.3
TLS otomatik: --core-tls-generate-cert
TLS manuel: --core-tls-crt <dosya> --core-tls-key <dosya>
Air-gapped mod: --air-gapped flag'ini ekle
Upgrade atla: --force-auto-upgrade
Preflight atla: --skip-preflight-check
Cluster oluşturma atla (sadece backend güncelle): --skip-cluster-creation
Single-node → multi-node geçiş: --multi-node-migrate --multi-node-enable --multi-node-ips <ip'ler>
Proxy: --core-proxy-host, --core-proxy-port, --core-proxy-user, --core-proxy-password, --core-proxy-non-proxy-hosts
SMTP: --core-smtp-host, --core-smtp-port (587), --core-smtp-from, --core-smtp-user, --core-smtp-password, --core-smtp-use-ssl, --core-smtp-start-tls
HPA varsayılan açık, kapatmak için: --core-enable-hpa=false

[down]
Cluster'ı durdurur. Container ve network bileşenlerini temizler, iptables kurallarını kaldırır.
Veri silinmez.
Kullanım: stanctl down
Flag yok, global flag'ler geçerli.

[agent]
Instana self-monitoring agent'ını yönetir.
Alt komutlar: apply, delete

[agent-apply]
Self-monitoring agent'ı kurar veya günceller.
Kullanım: stanctl agent apply [flags]
Alias: stanctl ag a
Mod: --agent-mode INFRASTRUCTURE | APM | AWS (varsayılan: INFRASTRUCTURE)
Endpoint: --agent-endpoint-host, --agent-endpoint-port (443)
Key: --agent-key
Zone/Cluster: --agent-zone-name (instana), --agent-cluster-name (instana)
Proxy: --agent-proxy-host, --agent-proxy-port (80), --agent-proxy-protocol (http|https), --agent-proxy-username, --agent-proxy-password, --agent-proxy-use-dns
Static image: --agent-use-static-image
Test (çalıştırmadan): --dry-run

[agent-delete]
Self-monitoring agent'ı siler.
Kullanım: stanctl agent delete
Alias: stanctl ag delete | del | rm | d
Flag yok.

[air-gapped]
Air-gapped (internetsiz) ortam için paket oluşturur veya içe aktarır.
Alt komutlar: package, import
Alias: stanctl agap

[air-gapped-package]
Air-gapped paketi oluşturur (internet erişimi olan makinede çalıştırılır).
Kullanım: stanctl air-gapped package [flags]
Sürüm belirt: --instana-release <sayı> VEYA --instana-version <versiyon> (ikisi birden kullanılmaz)
Çıktı dizini: -o, --output-dir <dizin>
Lisans: -l, --license-file <dosya>
Platform: -p, --platform linux/amd64 (varsayılan)
Cache kapatma: --no-cache
Registry ve Helm flag'leri gerekli (--download-key veya --registry-user/password).

[air-gapped-import]
Air-gapped paketini sisteme aktarır (hedef makinede çalıştırılır).
Kullanım: stanctl air-gapped import -f <paket-dosyası> [flags]
Zorunlu: -f, --file <arsiv-dosyası>
Opsiyonel: --cluster-data-dir (K8s ve Docker image dizini)

[auth]
Kimlik doğrulama yönetimi.
Alt komutlar: create-admin-api-token, reset-2fa, reset-idp, reset-password

[auth-create-admin-api-token]
Admin API token oluşturur.
Kullanım: stanctl auth create-admin-api-token
Flag yok.

[auth-reset-2fa]
Kullanıcının iki faktörlü kimlik doğrulamasını sıfırlar.
Kullanım: stanctl auth reset-2fa -e <email>
Zorunlu: -e, --email <kullanıcı-email>

[auth-reset-idp]
IDP (Identity Provider) yapılandırmasını sıfırlar.
Instana'nın dahili kullanıcı yönetimine geri döner.
Kullanım: stanctl auth reset-idp
Flag yok.

[auth-reset-password]
Kullanıcı şifresini sıfırlar. Şifre flag ile verilmezse interaktif sorulur.
Kullanım: stanctl auth reset-password -e <email> [-p <yeni-şifre>]
Zorunlu: -e, --email <kullanıcı-email>
Opsiyonel: -p, --password <yeni-şifre>

[backend]
Instana backend kurulumunu yönetir.
Alt komut: apply
Alias: stanctl be

[backend-apply]
Instana backend'i kurar veya günceller (cluster zaten varsa kullan).
Kullanım: stanctl backend apply [flags]
Alias: stanctl be a
up komutundaki tüm --core-* ve --unit-* flag'leri burada da geçerli.
Test (çalıştırmadan): --dry-run
Preflight atla: --skip-preflight-check
Air-gapped: --air-gapped

[benchmark]
Depolama performans testleri.
Alt komut: fio
Alias: stanctl bm

[benchmark-fio]
FIO storage benchmark çalıştırır. Dizin belirtilmezse tüm volume'lar test edilir.
Varsayılan test dizinleri: data, metrics, analytics, objects
Kullanım: stanctl benchmark fio [flags]
Tek dizin: --directory <yol>
Multi-node: --multi-node-enable --multi-node-ips <ip'ler>
Volume path'leri override: --volume-data, --volume-metrics, --volume-analytics, --volume-objects

[cluster]
K8s cluster oluşturma, silme, backup ve restore.
Alt komutlar: create, delete, backup, restore
Alias: stanctl cl

[cluster-create]
K8s cluster'ı oluşturur.
Kullanım: stanctl cluster create [flags]
Alias: stanctl cl c
Multi-node: --multi-node-enable --multi-node-ips <ip1,ip2,ip3>
Air-gapped: --air-gapped --cluster-data-dir <dizin>
Preflight atla: --skip-preflight-check
Registry ve cluster download flag'leri gerekebilir (--download-key).

[cluster-delete]
K8s cluster'ı siler. Varsayılan olarak veriyi de siler.
Kullanım: stanctl cluster delete [flags]
Alias: stanctl cl delete | del | rm | d
Veriyi koru: --keep-data (storage volume ve kullanıcı config'i siler)

[cluster-backup]
Cluster'ı yedekler.
Kullanım: stanctl cluster backup [-o <dizin>]
Çıktı dizini: -o, --output-dir (varsayılan: mevcut dizin ".")

[cluster-restore]
Cluster'ı yedekten geri yükler.
Kullanım: stanctl cluster restore -f <backup-dosyası> [flags]
Zorunlu: -f, --file <backup-arsiv-dosyası>
Multi-node restore: --multi-node-enable --multi-node-ips <ip'ler>
Air-gapped: --air-gapped --cluster-data-dir <dizin>

[datastores]
Instana'nın kullandığı veri depolarını yönetir (Cassandra, Elasticsearch, Kafka, ClickHouse, Postgres, BeeInstana).
Alt komut: apply
Alias: stanctl ds
Belirli datastore: -i, --includes <datastore-adı> (birden fazla: -i cassandra -i kafka)
Test: --dry-run

[datastores-apply]
Datastore'ları kurar veya günceller.
Kullanım: stanctl datastores apply [flags]
Alias: stanctl ds a
Belirli datastore: -i cassandra | elasticsearch | kafka | clickhouse | postgres | beeinstana
Her datastore için --<isim>-user, --<isim>-password, --<isim>-admin-user, --<isim>-admin-password flag'leri var.
Şifreler belirtilmezse otomatik üretilir.
Air-gapped: --air-gapped
Preflight atla: --skip-preflight-check

[debug]
Debug paketi oluşturur (container logları, manifest, sistem bilgisi, healthcheck, metrikler, config).
Kullanım: stanctl debug [-o <dizin>] [-n <namespace'ler>]
Çıktı dizini: -o, --output-dir
Namespace filtre: -n, --namespaces namespace1,namespace2
Sıkıştırma kapatma: --no-compress

[dump]
Yapılandırma verisini dışa aktarır.
Alt komut: config-data
Alias: stanctl dm

[dump-config-data]
Config verisini başka bir Standard Edition kurulumuna aktarmak için dump eder.
Kullanım: stanctl dump config-data [-o <dizin>]
Çıktı dizini: -o, --output-dir (varsayılan: mevcut dizin ".")

[license]
Instana lisans yönetimi.
Alt komutlar: download, info, update
Alias: stanctl lc

[license-download]
Lisans dosyasını indirir.
Kullanım: stanctl license download [--sales-key <key>] [-o <dizin>]
Sales key: --sales-key (veya STANCTL_SALES_KEY env)
Çıktı: -o, --output-dir (varsayılan: /root/.stanctl/license.json)

[license-info]
Mevcut lisansın geçerlilik bilgisini gösterir.
Kullanım: stanctl license info
Flag yok.

[license-update]
Lisansı, download key'i ve sales key'i günceller.
Kullanım: stanctl license update [flags]
Online: --download-key ve/veya --sales-key
Air-gapped (dosyadan): -f, --file <lisans-dosyası>

[migrate]
Self-Hosted Classic edition'dan veri göçü yapar.
Kullanım: stanctl migrate -f <arsiv-dosyası> [flags]
Alias: stanctl mg
Zorunlu: -f, --file <migrasyon-data-arsivi>
Install type: --install-type demo | production
Sürüm: --instana-release VEYA --instana-version
Multi-node: --multi-node-enable --multi-node-ips <ip'ler>
Volume path'leri override edilebilir.
Postgres, registry, Helm flag'leri gerekebilir.

[versions]
Instana versiyon ve image bilgilerini listeler.
Alt komutlar: identify, list-images
Alias: stanctl v

[versions-identify]
Mevcut kurulumla uyumlu Instana versiyonlarını listeler.
Kullanım: stanctl versions identify [flags]
Alias: stanctl v id
Registry erişimi gerekli: --download-key veya --registry-user/password/url

[versions-list-images]
Belirli bir Instana release'inin image listesini gösterir.
Kullanım: stanctl versions list-images --instana-version <versiyon> [flags]
Alias: stanctl v ls
Sürüm: --instana-version <versiyon>
Registry erişimi: --download-key veya --registry-user/password/url
