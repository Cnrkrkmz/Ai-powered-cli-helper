# stanctl — Backend and datastore management

[backend-apply]
Installs or updates the Instana backend (use when cluster is already running).
Usage: stanctl backend apply [flags]
Alias: stanctl be a
All --core-* and --unit-* flags from the up command are also valid here.
Dry run: --dry-run
Skip preflight: --skip-preflight-check
Air-gapped: --air-gapped
Key flags:
  --core-base-domain <domain>
  --download-key <key>
  --install-type demo | production
  --instana-release <number> OR --instana-version <version> (mutually exclusive)

[datastores-apply]
Installs or updates datastores used by Instana.
Usage: stanctl datastores apply [flags]
Alias: stanctl ds a
Specific datastore: -i cassandra | elasticsearch | kafka | clickhouse | postgres | beeinstana
Multiple: stanctl datastores apply -i cassandra -i kafka
Flag format per datastore: --<name>-user, --<name>-password, --<name>-admin-user, --<name>-admin-password
Passwords are auto-generated if not provided.
Air-gapped: --air-gapped
Dry run: --dry-run (global flag)

[migrate]
Migrates data from Self-Hosted Classic edition.
Usage: stanctl migrate -f <archive-file> [flags]
Alias: stanctl mg
Required: -f, --file <migration-archive>
Install type: --install-type demo | production
Version: --instana-release OR --instana-version
Multi-node: --multi-node-enable --multi-node-ips <ips>
Volume paths: --volume-data, --volume-metrics, --volume-analytics, --volume-objects
